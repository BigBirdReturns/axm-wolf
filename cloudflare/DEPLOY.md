# WOLF on the Glass Onion

This package keeps the existing direct-upload workflow. The assembled `Glass_Onion` directory contains the original site, the compiled WOLF app at `/wolf/`, and a prebuilt root `_worker.js`.

Before the first hosted deployment:

1. Create a D1 database.
2. Execute `schema.sql` against it.
3. On the existing Pages project, bind that database as `WOLF_DB`.
4. Add an encrypted Pages environment variable named `WOLF_ADMIN_KEY` with a long random value.
5. Upload the complete assembled `Glass_Onion` directory through the existing Cloudflare Pages direct-upload screen.

The root worker passes every non-WOLF request directly to `env.ASSETS`. It handles `/wolf/api/*` and rewrites `/wolf/dashboard` and `/wolf/SUR##` to the WOLF application shell.

The dashboard key is entered at `https://axm.tools/wolf/dashboard`. It is retained only in that browser tab. Recipient capability tokens are generated by the worker, hashed before D1 storage, and placed after `#k=` in invitation URLs so ordinary HTTP request logs do not receive them.
