§ 01

What Wolf Is

Wolf is a capture engine for institutional knowledge that exists only in someone's memory. A capture pack defines a question library — sections, prompts, lenses. The subject answers in their own words, in any order, at their own pace. The engine stores what was said, verbatim, and nothing more.

The pack is the question contract. The record is the source of truth. The engine never touches either with inference.

Local-First

All records, drafts, and installed packs live in this browser's IndexedDB. Nothing is sent to a server. There is no account and no hosted backend.

Offline

Installable as a PWA. After the first load, the app shell, the bundled pack, and core capture / search / export functions work without a network connection.

No AI at Runtime

v0.1 is not an AI interviewer. It asks fixed prompts from a pack and stores what the subject types or dictates. No summarization, no generated follow-ups, no embeddings.

Portable by Default

Capture packs (*.wolfpack.json) and records (*.wolfrecord.json) are plain, validated JSON. Export and import are designed to be lossless.

§ 02

The First Record

The bundled first pack is The Wolf's Deposition, a primary-source record for William Sandhu's forty-year career in institutional IT, framed as a Pulp-Fiction-inspired deposition. It currently defines 7 sections and 58 prompts, derived at runtime — no UI string hard-codes these counts.

The legacy proof of concept claimed "62 questions, 7 eras," but its source array contained only 58 prompt objects. The 58 prompts are the content authority: no prompts were invented to reach 62. The mismatch and the full migration to stable semantic prompt IDs are documented in reference/legacy-v0/AUDIT.md.

The pack frames its subject deliberately: a forty-year career, preserved as primary source, in the subject's own words.

§ 03

Three Layers, Small

The AXM protocol describes knowledge, journal, and hot-buffer layers that share one sealed container format. Wolf is not Genesis and does not seal shards — but the same temporal shape recurs at engine scale. The pack is reference material. Committed revisions are the append-only record of what was said. Drafts are working memory that never gets confused with testimony.

LayerWolf EquivalentMutabilityPurpose
Knowledge Capture pack (*.wolfpack.json), digested via packDigest Immutable The question contract: sections, prompts, lenses
Journal Committed response revisions Append-only, immutable per revision Testimony with provenance — what was said, and when
Hot Buffer Drafts (autosaved) Append-only, working memory In-progress answers, never confused with the committed record
§ 04

Guarantees

These follow directly from the design invariants (DESIGN.md Parts 1-2 and 12).

  • Packs are inert JSON: validated on import, no HTML, no executable content.
  • Edits append revisions. The record is never overwritten.
  • Export and import of records and packs is designed to be lossless.
  • Everything works offline after the first load — app shell, bundled pack, capture, search, export.
  • Progress counts (sections, prompts, completion) are always derived at runtime, never hard-coded.
  • No telemetry, no remote fonts, no network requests at runtime.
§ 05

Portable Formats

Two file formats carry everything Wolf produces and consumes. Both are plain JSON — readable, diffable, and not tied to this application.

*.wolfpack.json ← the question contract
  packId, packVersion
  sections[] → prompts[]
  lenses, presentation defaults
  packDigest ← SHA-256 over canonical JSON
 
*.wolfrecord.json ← the testimony
  recordId, packId, packVersion
  pack snapshot ← travels with the record
  responses[] → revisions[]
  provenance ← pack/app/engine versions, timestamps
Pack digest. packDigest is a SHA-256 hash over the canonicalized pack JSON (canonicalizePack, digestPack). It is the provenance anchor for every record built against that pack.
Honest limit. A digest detects tampering; it does not prove authorship. Signed packs through Genesis suites are the roadmap, not the present (DESIGN.md 12.5, 18.1).
§ 06

Quick Start

Run Locally bash
$ git clone https://github.com/BigBirdReturns/axm-wolf.git
$ cd axm-wolf
$ npm ci
 
# lint (engine boundary), typecheck, tests, build
$ npm run check
 
# local dev server
$ npm run dev
Deploy Modes build-time env
# platform (default): record library + pack library first
$ npm run build
 
# single-pack: emphasize one pack on launch
$ VITE_DEPLOY_MODE=single-pack VITE_DEFAULT_PACK_ID=wolfs-deposition npm run build
